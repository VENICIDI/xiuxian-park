# 修仙游乐园（Xian Park）MVP 文档

本文件定义 Game Jam 版本的最小可玩产品（MVP）范围，配合 [游戏设计文档.md](游戏设计文档.md)、[技术设计文档.md](技术设计文档.md) 与 [美术资产清单.md](美术资产清单.md) 使用。目标是明确"做什么、不做什么、以及做完的标准"。

## 1. MVP 目标

一句话：玩家能在 PC 浏览器中，从新游戏开始，连续经营多天乐园，通过摆放与联动建筑赚取灵石，并在失败或终局时看到结算。

核心验收（Definition of Done）：

- 从新游戏可完整游玩到失败或终局。
- 建筑放置、联动、游客营业、结算、三选一、下一天闭环可运行。
- 不同摆放方案产生明显且可解释的收益差异。
- 刷新页面后能从正确阶段继续（本地存档）。
- 生产构建可通过静态 URL 打开并游玩，核心流程无阻断问题。

## 2. 实现边界

### 2.1 范围内（In Scope）

- 单机、纯前端、静态部署，无后端与账号。
- 固定 8×6 地图，共 48 格，单张地图 `park-01`。
- 约 15 个建筑，数据驱动配置。
- 6 个游客门派，带偏好差异。
- 游客沿固定路线移动（按 Tiled `order` 排序），无寻路 AI。
- 一天状态机：Planning → Resolving → Animating → Drafting。
- 确定性随机（统一 PRNG + seed），结算可复现。
- "先结算后播放"：规则同步计算结果 + 事件日志，动画只做表现。
- 本地存档（localStorage，带 schemaVersion）。
- Phaser Sound 音频（音乐 + 基础音效）。
- PC 浏览器横屏，设计分辨率 1280×720。

### 2.2 范围外（Out of Scope，MVP 不做）

- 多人 / 联机 / 云存档 / 排行榜。
- 移动端响应式、触控手势、PWA 离线安装。
- 多地图、可变入口出口、地图编辑器内置化。
- A* 或动态寻路。
- React/Vue 等 UI 框架、Redux/Zustand、ECS 框架。
- Howler 双音频栈（除非 Phaser Sound 出现无法规避问题）。
- Lottie 动画（可选增强，时间不足直接删除）。
- 复杂教程系统、剧情、成就系统。
- 运行时热更新配置 / JSON 外部化（开发期用 TS 配置）。

### 2.3 技术约束

- 规则层不得引用 `Phaser`，保证可单元测试与可复现。
- 建筑效果使用有限可辨识联合类型（EffectSpec），不写任意回调。
- 所有游戏随机走统一 `RandomService`，禁止直接 `Math.random()`。
- `board` 固定为 48 项一维数组，索引 `y * 8 + x`。
- 60 FPS 目标；同屏游客 ≤ 40；单日纯规则结算 < 16 ms。

## 3. 实现功能点

按模块拆分，均为 MVP 必做项。

### 3.1 地图与放置

- [ ] 加载 Tiled 地图，渲染地面、道路、入口、出口。
- [ ] 渲染 8×6 网格与可放置/不可放置区分。
- [ ] 建筑选择进入放置模式。
- [ ] 鼠标悬停显示合法/非法落点预览。
- [ ] 点击合法格放置并扣灵石；非法格拒绝。
- [ ] 点击已有建筑打开详情，可升级或取消。
- [ ] 灵石不足时禁用购买/升级。

### 3.2 游戏状态与回合

- [ ] `GameState` 数据结构与阶段状态机。
- [ ] Planning 阶段锁定/解锁棋盘编辑。
- [ ] "开始营业"触发同步结算。
- [ ] Resolving 生成结算快照 + 应用每日事件。
- [ ] Animating 按事件日志播放，支持 1×/2×/跳过。
- [ ] Drafting 三选一后进入下一天并自动存档。
- [ ] 天数推进与失败/终局判定。

### 3.3 联动与经济

- [ ] 固定收益计算顺序（快照 → 全局标签加成 → 相邻加成 → 游客模拟 → 结算后效果）。
- [ ] 加法汇总后一次相乘，稀有"翻倍"进独立乘区。
- [ ] 聚灵阵（相邻 +15%）、雷池（雷系全局加成）、悟道台（+1 停留）。
- [ ] 护山大阵（免疫负面事件）、渡劫保险（失败部分收益）。
- [ ] 孟婆奶茶（恢复疲劳）、法宝纪念馆（高满意度额外消费）、仙丹铺（基础消费）。
- [ ] 九转大摆锤（相邻刺激越多越高）、黄泉漂流（路径越长越高）。
- [ ] 建筑升级（Lv1-Lv4 数值提升）与品质系数（白/绿/蓝/紫/橙）。

### 3.4 游客

- [ ] 6 门派生成与属性（钱包/满意度/刺激/疲劳/停留次数）。
- [ ] 沿固定路线移动并逐点触发建筑。
- [ ] 门派偏好影响消费与满意度（如佛修厌恶过度刺激、魔修喜刺激）。
- [ ] 重复消费概率（灵酒馆/相关效果）。

### 3.5 Roguelike 与事件

- [ ] 每天结束三选一建筑。
- [ ] 每日事件（天降祥瑞、雷劫提前、宗门团建、魔修夜场、仙盟检查、炼丹炸炉）。
- [ ] 招生广场（下一天游客数量增加）。

### 3.6 表现与 UI

- [ ] HUD：天数、灵石、游客数、当前事件。
- [ ] 建筑手牌 / 卡片（含品质边框、价格、预计收益）。
- [ ] 收益飘字、消费气泡、雷击等关键特效。
- [ ] 结算界面：天数、总收入、最高连携、失败原因。
- [ ] 按钮四态（普通/悬停/按下/禁用）。

### 3.7 系统与基础设施

- [ ] 确定性 `RandomService`（seed + 游标）。
- [ ] 版本化本地存档（放置后/结算后/三选一后/页面隐藏时）。
- [ ] 存档损坏或版本不兼容时不覆盖，提示并允许新开局。
- [ ] `AudioService`（music/sfx 双通道，首次交互后启动）。
- [ ] 开发模式调试面板（seed、阶段、收益明细、加灵石、清存档）。

## 4. MVP 美术清单（最小子集）

完整清单见 [美术资产清单.md](美术资产清单.md)，此处为 MVP 必需最小集与文件类型。占位美术（纯色块/简笔）即可先跑通玩法，正式美术后置。

### 4.1 建筑（15 个，图集 buildings.png + buildings.json）

- 游乐/商店 8 个：`idle` + `active` + `highlight`（可禁用者加 `disabled`）。
  - sword-coaster, spin-hammer, thunder-tower, heart-demon-house, yellow-spring-drift, mengpo-tea, pill-shop, artifact-museum
- Buff/功能 7 个：`idle` + `highlight`。
  - spirit-gathering-array, enlightenment-altar, thunder-pond, toilet-array, tribulation-insurance, recruit-plaza, mountain-guard-array

### 4.2 游客（6 门派，图集 visitors.png + visitors.json）

- `walk` 必需；`idle/happy/angry` 建议。单向绘制 + 水平翻转即可。
- sword, alchemy, buddhist, demonic, demon, ghost

### 4.3 特效（图集 effects.png + effects.json）

- 必需：fx-coin（收益）、fx-purchase（消费）、fx-thunder（雷系）、fx-place-valid、fx-place-invalid。
- 建议：fx-satisfaction-up、fx-ascension、fx-negative。

### 4.4 地图（Tiled）

- tileset-park.png（PNG）：地面/道路/装饰/可放置/不可放置。
- park-01.json（Tiled 导出 JSON）：含 Ground/Blocked/Route/Markers。

### 4.5 UI（图集 ui.png + ui.json）

- 按钮四态、面板、卡片底、进度条、Tooltip。
- 图标：灵石、游客、天数、满意度、倍速、跳过、音量。
- 品质边框 ×5（白/绿/蓝/紫/橙）。

### 4.6 场景大图与字体

- logo.png、bg-mainmenu.png、bg-preload.png、bg-result.png（PNG）。
- 中文主字体 main.woff2（WOFF2）。

### 4.7 文件类型速查

| 类别 | 运行时文件类型 |
| --- | --- |
| 图集（建筑/游客/特效/UI） | `.png` + `.json` |
| Tileset | `.png` |
| 地图 | `.json` |
| 场景大图/Logo | `.png` |
| 字体 | `.woff2` |
| 可选 Lottie | `.json` |

## 5. 里程碑（对应技术文档实施顺序）

| 阶段 | 内容 | 完成标准 |
| --- | --- | --- |
| M1 可点击棋盘 | Vite+TS+Phaser、占位地图、8×6 网格、放置 | 可稳定选择并放置占位建筑 |
| M2 一天可运行 | 状态机、固定路线、游客、基础收入、动画 | 可连续运行多天，收入可复现 |
| M3 核心联动 | Buff、门派偏好、15 建筑配置、加成展示 | 摆放差异产生可解释收益差 |
| M4 Roguelike | 三选一、每日事件、升级品质、失败与结算、存档 | 完整游玩到胜/负并可续玩 |
| M5 打磨 | 正式地图、图集、音频、倍速跳过、部署验收 | 生产 URL 可玩，无阻断问题 |

## 6. Game Jam 裁剪优先级

时间不足时按此顺序裁剪（越靠前越先砍）：

1. Lottie。
2. Howler（保留 Phaser Sound）。
3. 复杂设置页与非核心转场。
4. 游客动画种类（保留门派数值差异）。
5. 每日事件数量。
6. 建筑升级动画与品质特效。
7. 最后才动核心联动建筑。

不可裁剪：建筑放置、固定路线营业、可解释联动、收入与升级循环、每日三选一、至少一个明确终局条件。
