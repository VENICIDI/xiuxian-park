# 修仙游乐园（Xian Park）美术设计规范

本规范用于统一游戏内的**颜色、字体、圆角、按钮、卡片、图标、地图、动画反馈**等视觉标准，供美术与程序共同遵守。相关内容可与 [美术资产清单.md](美术资产清单.md)、[技术设计文档.md](技术设计文档.md) 配合使用。

> 核心原则：**统一、可爱（Q 版）、软色调、紫色 + 玉绿 + 金点缀、玩家一眼看懂。**

---

## 一、颜色规范

### 1.1 主色

| 色名 | 色值 | 用途 |
| --- | --- | --- |
| 灵气绿 | `#7FD37B` | 成功、收益、放置成功、按钮 |
| 灵石金 | `#FFD45C` | 金币、价格、奖励 |
| 修仙蓝 | `#69B7FF` | Buff、灵气、法阵 |
| 雷劫紫 | `#9C7DFF` | 稀有、传奇、雷系 |
| 危险红 | `#FF6767` | 扣钱、事件、失败 |

### 1.2 UI 背景

- 深紫灰：`#2E2348` 或 `#3A2F5E`

### 1.3 描边

- 描边色：`#FFFFFF30`（半透明白）
- **不要使用纯白描边。**

---

## 二、字体

- 推荐字体：**阿里妈妈方圆体**
- 备选字体：**HarmonyOS Sans**

| 层级 | 字号 |
| --- | --- |
| 标题 | 32 |
| 副标题 | 24 |
| 正文 | 18 |
| 说明 | 14 |

数字规则：

- 数字一律**粗体**。
- 金币数字**必须为黄色**（`#FFD45C`）。

---

## 三、圆角

- 全项目统一圆角：**16px**。
- **不要**出现按钮 16、面板 8、卡片 20 这类不一致情况——全部 16。

---

## 四、阴影

按钮阴影统一：

- Y 偏移：`4`
- 模糊 Blur：`12`
- 不透明度 Opacity：`20%`

**不要使用重阴影。**

---

## 五、按钮规范

| 类型 | 颜色 | 示例场景 |
| --- | --- | --- |
| 主按钮 | 绿色（`#7FD37B`） | 开始营业 |
| 次按钮 | 灰紫 | 关闭、取消 |
| 危险按钮 | 红色（`#FF6767`） | 拆除 |

主按钮示意：

```text
┌──────────────┐
     开始营业
└──────────────┘
```

交互状态：

- **Hover**：整体亮一点。
- **按下**：缩放至 **95%**。

---

## 六、建筑卡片

统一布局：**左图 + 右文字**。

```text
┌─────────────────┐
🏯 雷劫体验塔
⚡ 游乐设施
收益：20
价格：80
└─────────────────┘
```

品质决定**左侧竖条颜色**：

| 品质 | 竖条颜色 |
| --- | --- |
| 白（普通） | 灰 |
| 蓝（稀有） | 蓝 |
| 紫（史诗） | 紫 |
| 橙（传说） | 金 |

---

## 七、Icon 规范

- 风格统一：**线面结合**、**Q 版**。
- **不要**混用扁平 / 3D / 写实等多种风格。

常用图标示例：

| 含义 | Icon |
| --- | --- |
| 金币 | 💰 |
| 灵气 | ✨ |
| 游客 | 🙂 |
| 雷 | ⚡ |
| 满意 | 😊 |
| Buff | ⬆️ |

---

## 八、地图

- **不要**黑色格子。
- 建议使用**草地**底。
- Hover 格子时显示**绿色**高亮。

```text
□□□□
 ↓
🟩
```

放置提示：

- 可以放：**绿色**。
- 不能放：**红色**。

## 十、建筑标签

- **不要**一大段描述文字。
- 用短标签让玩家**一扫就懂**。

示例：

```text
🏪 商店
😊 回复疲劳
💰 +10 收入
```

---

## 十一、Buff 表现

- **不要**用文字表现 Buff。
- 建筑**左上角**用图标叠加显示，例如：`↑15%` 或 `✨`。
- **Hover** 时再显示详情。

---

## 十二、游客 UI

- 游客头顶最多显示**两个 Icon**。
- **不要**一堆文字。

示例：

```text
🙂
💰
😊
```

---

## 十三、数字动画

| 类型 | 颜色 | 示例 | 表现 |
| --- | --- | --- | --- |
| 赚钱 | 黄色 | `+20` | 向上飘、消失 |
| Buff | 绿色 | `+15%` | 向上飘、消失 |
| 失败 | 红色 | `-30` | 向上飘、消失 |

---

## 十四、建筑放置反馈

放下建筑时**立即 POP**：

- 缩放序列：`0.8 → 1.05 → 1`
- 随后：**灵气扩散**特效。

---

## 十五、营业反馈（最重要）

营业时，**整个地图要一直"活着"**，让玩家不停看到数字反馈：

```text
🙂  →  🏯  →  💰  →  😊  →  ✨
```

即：游客进入 → 使用建筑 → 产出金币 → 满意 → 灵气特效，循环不断。

## 十七、资源尺寸规范（程序与美术必须统一）

> Game Jam 最容易出 Bug 的地方，建议**第一天就锁死**。

| 资源 | 尺寸 | 备注 |
| --- | --- | --- |
| 地图格子 | 96 × 96 px | 视觉更舒服，适合放建筑 |
| 小图标（Icon） | 32 × 32 px | 灵石、游客、Buff 等 |
| 建筑缩略图 | 64 × 64 px | 建筑列表使用 |
| 建筑实体 | 160 × 160 px（约占 1×1 格视觉） | 底部对齐格子中心 |
| NPC | 64 × 64 px | Sprite Sheet 单帧 |
| 按钮高度 | 56 px | 移动端也容易点击 |
| 卡片圆角 | 16 px | 全项目统一 |
| 页面边距 | 24 px | 保持呼吸感 |

---

## 十八、UI 层级（Z-Index）

程序按固定层管理，避免人物跑到建筑后面、特效被 UI 遮挡等问题：

| Layer | 内容 |
| --- | --- |
| Layer 0 | 背景（草地、池塘） |
| Layer 1 | 道路 |
| Layer 2 | 建筑阴影 |
| Layer 3 | 建筑 |
| Layer 4 | 游客 |
| Layer 5 | 粒子特效（金币、灵气） |
| Layer 6 | 格子高亮 |
| Layer 7 | HUD（顶部状态栏） |
| Layer 8 | 弹窗、Tooltip |
| Layer 9 | 全屏特效（结算、升级） |
