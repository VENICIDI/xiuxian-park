# 修仙游乐园（Xian Park）美术资产清单

本清单基于 [游戏设计文档.md](游戏设计文档.md) 与 [技术设计文档.md](技术设计文档.md)，用于统一美术产出内容、文件类型和存放位置。

## 1. 通用规范

### 1.1 像素与尺寸

- 逻辑 Tile：`96 × 96` 像素。
- 推荐原始像素单元：`32 × 32`，运行时按 3 倍整数缩放。
- 建筑默认占地：1×1 Tile。
- 建筑锚点：底部居中。
- 统一调色板，避免每张图独立配色。

### 1.2 文件类型约定

| 用途 | 源文件类型 | 导出/运行时类型 |
| --- | --- | --- |
| 像素图（角色/建筑/特效/图标） | `.aseprite` | `.png` + `.json`（图集） |
| 地图 | `.tmx`（Tiled 工程） | `.json`（Tiled 导出） |
| Tileset 图片 | `.aseprite` / `.png` | `.png` |
| UI 大图/背景 | `.aseprite` / `.psd` | `.png` |
| 矢量/UI 动画（可选） | `.aep` / Lottie 工程 | `.json`（Lottie） |
| 字体 | `.ttf` / `.otf` | `.woff2`（Web 优化） |

说明：

- 图集数据文件项目内统一使用一种格式（JSON Array 或 Hash）。
- 源 `.aseprite`、`.tmx`、`.psd` 放在仓库 `art/` 目录，不进入运行时加载。
- 运行时导出物放入 `public/assets/`。
- 所有位图导出为 32 位 PNG（含 Alpha）；不使用 JPG（避免像素边缘失真）。

### 1.3 存放位置

```text
art/                         # 源文件（不打包）
├─ aseprite/
├─ tiled/
├─ ui/
└─ lottie/
public/assets/               # 运行时资产（打包）
├─ maps/                     # .json（Tiled 导出）
├─ tilesets/                 # .png
├─ atlases/                  # .png + .json
├─ ui/                       # .png
├─ fonts/                    # .woff2
└─ lottie/                   # .json（可选）
```

### 1.4 动画标签（Aseprite Tag）

统一英文标识：`idle`、`active`、`disabled`、`highlight`、`walk`、`happy`、`angry`。

### 1.5 命名规范

- Frame 名称：`类型/资源-id/动画-序号`，例如 `building/thunder-tower/active-0`。
- 文件名使用小写中划线，例如 `thunder-tower.png`。

## 10. 交付优先级

1. 建筑 `idle` + 放置高亮（保证可玩）。
2. 游客 `walk` + 地图 Tileset。
3. 灵石飘字、消费气泡、雷击特效。
4. 基础 UI（按钮、面板、卡片、图标）。
5. 建筑 `active`/`disabled` 与品质边框。
6. 场景大图、Logo、字体。
7. 可选 Lottie 与额外特效打磨。

## 11. 逐帧命名清单

Frame 命名统一格式：`类型/资源-id/动画-序号`，序号从 `0` 开始。以下列出各图集的 frame key 规则与示例；实际帧数由动画长度决定，此处给出 MVP 建议帧数。

### 11.1 建筑图集（buildings.png / buildings.json）

规则：`building/{id}/{tag}-{n}`。

| 动画 tag | 建议帧数 | 示例 frame key |
| --- | --- | --- |
| idle | 1-4 | `building/thunder-tower/idle-0` |
| active | 4-8 | `building/thunder-tower/active-0` … `active-7` |
| disabled | 1 | `building/thunder-tower/disabled-0` |
| highlight | 1-2 | `building/thunder-tower/highlight-0` |

完整建筑 id 列表（用于批量生成 frame key）：

```text
sword-coaster / spin-hammer / thunder-tower / heart-demon-house / yellow-spring-drift
mengpo-tea / pill-shop / artifact-museum
spirit-gathering-array / enlightenment-altar / thunder-pond
toilet-array / tribulation-insurance / recruit-plaza / mountain-guard-array
```

### 11.2 游客图集（visitors.png / visitors.json）

规则：`visitor/{id}/{tag}-{n}`。

| 动画 tag | 建议帧数 | 示例 frame key |
| --- | --- | --- |
| walk | 2-4 | `visitor/sword/walk-0` … `walk-3` |
| idle | 1-2 | `visitor/sword/idle-0` |
| happy | 2 | `visitor/sword/happy-0` |
| angry | 2 | `visitor/sword/angry-0` |

门派 id：`sword / alchemy / buddhist / demonic / demon / ghost`。

若采用单向 + 水平翻转，则只需绘制向右方向，运行时用 `setFlipX(true)` 表现向左。

### 11.3 特效图集（effects.png / effects.json）

规则：`fx/{id}/play-{n}`。

| 特效 id | 建议帧数 | 示例 frame key |
| --- | --- | --- |
| fx-thunder | 4-6 | `fx/fx-thunder/play-0` |
| fx-coin | 1-2 | `fx/fx-coin/play-0` |
| fx-purchase | 2-4 | `fx/fx-purchase/play-0` |
| fx-satisfaction-up | 2-4 | `fx/fx-satisfaction-up/play-0` |
| fx-ascension | 6-8 | `fx/fx-ascension/play-0` |
| fx-negative | 4 | `fx/fx-negative/play-0` |
| fx-place-valid | 1 | `fx/fx-place-valid/play-0` |
| fx-place-invalid | 1 | `fx/fx-place-invalid/play-0` |

### 11.4 UI 图集（ui.png / ui.json）

规则：`ui/{id}[/{state}]`。状态用于按钮四态。

| 资产 | frame key |
| --- | --- |
| 按钮四态 | `ui/button/normal`、`ui/button/hover`、`ui/button/pressed`、`ui/button/disabled` |
| 面板 | `ui/panel` |
| 卡片底 | `ui/card` |
| 品质边框 | `ui/rarity/common`、`.../uncommon`、`.../rare`、`.../epic`、`.../legendary` |
| 图标 | `ui/icon/stone`、`ui/icon/visitor`、`ui/icon/day`、`ui/icon/satisfaction` |
| 控制图标 | `ui/icon/speed`、`ui/icon/skip`、`ui/icon/volume` |
| 进度条 | `ui/progress/bg`、`ui/progress/fill` |
| Tooltip | `ui/tooltip` |
